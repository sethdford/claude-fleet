FAQ

Frequently asked questions

What is Claude Fleet?

Claude Fleet is an orchestration layer for running multiple Claude Code CLI instances in parallel. It provides coordination, task management, and real-time monitoring for multi-agent workflows.

Do I need an Anthropic API key?

Yes, each Claude Code worker requires access to the Anthropic API through your existing Claude Code CLI setup. Claude Fleet orchestrates the workers but doesn't handle API authentication directly.

How does git worktree isolation work?

Each worker operates in its own git worktree, allowing them to work on different branches simultaneously without conflicts. Changes are merged back through standard git workflows.

Can I use this in production?

Claude Fleet is designed for production use with SQLite persistence, automatic crash recovery, health monitoring, and Prometheus metrics. It's MIT licensed for any use case.

How many workers can I run?

The default limit is 5 concurrent workers, configurable via MAX_WORKERS. Actual limits depend on your system resources and Anthropic API rate limits.

What's the MCP integration for?

The Model Context Protocol server lets any MCP-compatible client (including Claude Code itself) control the fleet, enabling agents to spawn and coordinate other agents.
